Introduction

The Portfolio Mapping Tool 3.0 (PMT 3.0) is a web-based mapping application intended to visualize development work in a variety of ways. The features of this web application allow users to browse, map, add, edit, and share projects without the need for geographic information systems expertise. This User Guide provides the basic information necessary to understand how to use the PMT 3.0.

Purpose

The PMT 3.0 is ultimately intended to help users improve project strategies and partnerships for greater impact in their work. Its features and functions have been designed to provide the following benefits:

  • Inform strategic and project management decisions. The PMT 3.0 can help inform decisions by allowing users to take geographic information into account, whether it is the location of markets, related projects and partners, travel time, annual precipitation, or maize crop yields.
  • Communicate programmatic projects to key stakeholders. A primary benefit to users of the PMT 3.0 is to see the spatial layout of their projects relative to context. Users have the ability to add their projects to the PMT 3.0 database and then visualize those projects in a variety of ways.
  • Understand how programmatic efforts relate to other projects as well as to useful agricultural information. Users have the ability to browse and map other people’s projects alone and alongside their own projects. This functionality provides the framework for multiple organizations to communicate vital strategic information together in a coordinated fashion.
The PMT 3.0 is designed to be flexible and expandable, allowing users to develop unique ways of applying the tool to their work.

Features and Capabilities

The PMT 3.0 enables users to find projects, make maps with those projects, and enter new project data. Each of these primary features is described in greater detail in corresponding sections of this User Guide.

  • Summaries Module: Users are presented with a set of project filters and summary visualizations that characterize the project information.
    • Filter by: Region, Agricultural Commercialization Clusters (ACC), Activity Status and Date Range
    • National Summaries: Top Funding Partners by Investment, Share of Investments by Program, Project Investments Matrix, Project Count by Crop, Project Count by Livestock and Project Counts by Investment Sector
    • Regional Summaries: Top Projects, Projects by Program, Projects by Sector Category, Project Investments Matrix, Project County by Crop, Project Count by Livestock and Project Counts by Investment Sector
    • Export data: Download graphic or export data into csv format for use offline
  • Projects Module: Users have the ability view, filter and explore the project information in detail from a tabular perspective.
    • Filter by: Keyword (free text), Funding Partners, Implementers, Program, Sector Category, Sector, Thematic Area, Crops, Livestock, Women in Development, Activity Status, National or Local and by UNOCHA Boundaries
    • Sort by: Project Name, Budget, Start Date, End Date or Funding Partner
    • View project details: Description, Project Footprint, Locations, Contacts, Dates, Investment Information, Taxonomies and Participating Organizations (funding, implementing, accountable)
    • Export data: Export data into csv format for use offline
  • Map Module: Users have the ability view, filter and explore the project information in detail from a spatial perspective, in connection with other spatially related information.
    • Filter by: Keyword (free text), Funding Partners, Implementing Organizations, Program, Sector Category, Sector, Thematic Area, Crops, Livestock, Women in Development, Activity Status, National or Local and by UNOCHA Boundaries
    • Cluster by: Regions or Agricultural Commercialization Clusters (ACC)
    • View project details: Description, Locations, Contacts, Dates, Investment Information, Taxonomies and Participating Organizations (funding, implementing, accountable)
    • Map Overlays: Institutional Surveys, Elevation, Rainfall, Crop Statistics, Demographics, Land Use and Soil Resources.
    • Reference Layers: Change basemaps, add region and/or district administrative boundaries
    • Tools: Search for a geographic location and Walkshed Tool
    • Export data: Export data into csv format for use offline
  • Editor Module: Users have the ability to create new projects, edit existing project information or delete projects. This feature requires a login with appropriate rights.
    • Project management: Create new project, edit existing project information (financials, organizations, taxonomies, locations), delete project.
  • Administrative Console: Users have the ability manage user access and permissions and perform routine data management tasks. This feature requires a login with appropriate rights.
    • User management: Create new user, grant/revoke access to existing user, delete users, update user role, grant editing permissions to projects, reset password.
    • Data management: Rebuild database views.
    • Login: View user's last login information
  • Summaries Module: Users are presented with a global investment summary with the option to explore individual country's grant filters and summary visualizations that characterize the grant information.
    • Filter by: Date Range
    • National Summaries: Top Funding Partners by Investment, Share of Investments by Program, Agricultural Investments Matrix, Grant Count by Crop, Grant Count by Sub-Program
    • Regional Summaries: Top Implementing Partners by Grant Count, Grants by Program, Grants by Sub-Program, Agricultural Grants by Implementer Matrix, Grant Count by Crop, Grant Count by Sub-Program
    • Export data: Download graphic or export data into csv format for use offline
  • Grants Module: Users have the ability view, filter and explore the grant information in detail from a tabular perspective.
    • Filter by: Keyword (free text), Donors, Partners, Program, Sub-Program, Focus Commodities, Grant Type
    • Sort by: Grant Name, Budget, Start Date, End Date or Donor
    • View project details: Description, Grant Footprint, Locations, Contacts, Dates, Investment Information, Taxonomies and Participating Organizations (donors & partners)
    • Export data: Export data into csv format for use offline
  • Map Module: Users have the ability view, filter and explore the grant information in detail from a spatial perspective, in connection with other spatially related information.
    • Filter by: Keyword (free text), Donors, Partners, Program, Sub-Program, Focus Commodities, Grant Type/li>
    • View project details: Description, Locations, Contacts, Dates, Investment Information, Taxonomies and Participating Organizations (donors & partners)
    • Map Overlays: Agro Dealers and Seed Companies.
    • Reference Layers: Multiple basemaps.
    • Tools: Search for a geographic location and Walkshed Tool
    • Export data: Export data into csv format for use offline
  • Integration Module: Users have the ability integrate data from the AGRA MIS into the AGRA PMT.
    • Integration: Initiate and view progress of integration between MIS and PMT.
  • Administrative Console: Users have the ability manage user access and permissions and perform routine data management tasks. This feature requires a login with appropriate rights.
    • User management: Create new user, grant/revoke access to existing user, delete users, update user role, grant editing permissions to projects, reset password.
    • Data management: Rebuild database views.
    • Login: View user's last login information

The web address for PMT 3.0 is http://v10.investmentmapping.org/ethaim/.

Access to the PMT 3.0 is open to the public, however there is a login option for features that require authenticated access. If you would like to request access to these features, please contact somebody from UDBS. Access is granted at the discretion of the University of Dar es Salaam Business School (UDBS).

Access to the PMT 3.0 is open to the public, however there is a login option for features that require authenticated access. Access is granted at the discretion of the AGRA.

The web address for PMT 3.0 is http://v10.investmentmapping.org/tanaim/.

Access to the PMT 3.0 is by invitation only. If you would like to request one, please do so via the link on the login page. Access is granted at the discretion of the Bill and Melinda Gates Foundation.

System Requirements

In order to access the PMT 3.0, you will need an internet connection with a minimum speed of 32k and a web browser. The recommended web browser for PMT 3.0 at this time is Chrome™, however the PMT 3.0 will work with other browsers such as Firefox™, Internet Explorer™ (version 11 or higher), and Safari™. The screen resolution for your computer should be set at a minimum of 1024 x 768.

Logging In to PMT 3.0

The PMT 3.0 is open to the public. However, in order to get access to restricted functionality like the Editor or Admin pages you will be required to login. To login, first click on the link in the upper right corner of the application header.

The login page will appear. Then simply enter your Username and Password into the corresponding boxes and click the LOG IN button.

The PMT 3.0 is an access controlled website. On the login page, simply enter your Username and Password into the corresponding boxes and click the LOG IN button. If your login is successful, you will land on the PMT 3.0 Summaries page.

User Information

Each user listed in the user list has a set of information associated to it.

First Name the first name of the user, this is required information when editing or adding new users.
Last Name the last name of the user, this is required information when editing or adding new users.
Email the user's email, this is required information when editing or adding new users. The email address is used to email the user when a new account is created or when their password has be updated.
Organization the organization in which the user belongs to, this is required information when editing or adding a new user.
Role the user's role in the application, which grants them privileges to features and functionality. This is required information when editing or creating new user. Below is a description of the four available roles and their privileges:
Reader role allows users only read access to the application and its data. Reader role does not have access to the Editor or Admin pages. If the application is open to the public then this role has no value for a user.
Editor role allows users access to the editor page, where user can edit or delete only those projects he/she has been granted permission to and add new projects. See the section on this page about editing a user's permissions for how to grant an Editor permissions to a project.
Administrator role allows users access to the editor page and the administrative page. User can edit any project, add new projects and delete any project. User can edit user information, add new users and delete users.
Super role allows access to all areas of the application with no exclusions to any functionality.
Last Login shows the last day and time the user logged in to the application. The message "User has never logged in" will show in place of a date if the user has never logged into the application.
Permissions shows a summary of the user's permissions based on their assigned role.

Locations Tab

The locations tab contains all the locations associated to a project. There can be one or more locations for a project. It is required that there is at least ONE location.

Editing Locations

To add a Region to the project click the add button add_circle to open the ADD REGIONS popup. To remove a region location, click the delete delete_forever icon adjacent to the region.

To add a Zone to the project click the add button add_circle to open the ADD ZONES popup. Zones are organized by their respective Regions. To remove a zone location, click the delete delete_forever icon adjacent to the zone.

To add a Woreda to the project click the add button add_circle to open the ADD WOREDAS popup. To filter the woredas, use the dropdown lists at the top of the window to filter the list to Region and Zone. To remove a woreda location, click the delete delete_forever icon adjacent to the woreda.

National Projects

To set a project as a national project, click the flag flag. The project location will then be set to National. A project can NOT have other locations if set to national. To remove a national project location, click the delete delete_forever icon.

Highlighting Locations

Click the highlight icon wb_sunny adjacent to any location to toggle the location's highlight on and off. Use the button in the upper right corner of the map to remove all highlighted locations.
